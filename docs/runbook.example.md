# LatteCam Runbook Example

Use this file as a template for your own local runbook. Do not commit your filled-in copy if it contains real IPs, passwords, HomeKit PINs, certificate serials, or Scrypted pairing state.

## Host

- Mac host name: `YOUR_MAC_HOST.local`
- Mac LAN IP: `YOUR_MAC_IP`
- MediaMTX config: `/opt/homebrew/etc/mediamtx/mediamtx.yml`
- Scrypted UI: `https://localhost:10443/`

## Secure Media Topology

```text
Old iPhone LatteCam
-> RTMPS rtmps://YOUR_MAC_IP:1936/live
-> MediaMTX
-> RTSP rtsp://127.0.0.1:8554/live/lattecam
-> Scrypted
-> HomeKit encrypted stream
-> Apple Home App
```

## iOS App Configuration

```text
Mac host or IP: YOUR_MAC_IP
RTMPS application: live
Stream name: lattecam
Publish username: lattecam_publish
Publish password: generated by scripts/render-mediamtx-config.sh
```

The publish password is stored in iOS Keychain. Do not store it in this runbook.

## MediaMTX

Recommended secure defaults:

- RTMPS only on `YOUR_MAC_IP:1936`.
- RTSP only on `127.0.0.1:8554`.
- HLS, WebRTC, SRT, and MoQ disabled unless explicitly needed.
- `overridePublisher: false`.
- Internal authentication enabled for `live/lattecam`.

## Scrypted

Add the RTSP Camera URL:

```text
rtsp://127.0.0.1:8554/live/lattecam
```

Expose the camera through the Scrypted HomeKit plugin in Bridge mode. Avoid standalone camera accessory mode unless you have verified it pairs reliably in your environment.

## Verification

```sh
scripts/verify-local-stack.sh YOUR_MAC_IP 1936 rtsp://127.0.0.1:8554/live/lattecam
```

Expected:

- MediaMTX listens on `YOUR_MAC_IP:1936`.
- MediaMTX listens on `127.0.0.1:8554`.
- Scrypted can read `live/lattecam` only after the iPhone is publishing.
- Apple Home App shows `LatteCam` under the Scrypted Bridge.
